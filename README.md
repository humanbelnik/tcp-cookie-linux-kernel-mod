# tcp-cookie-linux-kernel-mod

XDP-модуль ядра Linux, реализующий защиту сетевого интерфейса от DDoS-атаки SYN Flood с помощью TCP Cookies. Модуль перехватывает входящие кадры данных до того, как ядро выделяет ресурсы на обработку соединения.
 
## Принцип работы
 
Стандартное TCP-рукопожатие требует выделения памяти под структуры ядра уже на этапе получения SYN-запроса. Атака SYN Flood заполняет очередь полуоткрытых соединений (SYN Queue, по умолчанию 4096 записей), после чего сервер перестаёт обслуживать легитимных клиентов.
 
Модуль устраняет эту проблему следующим образом:
 
- При получении SYN-запроса ресурсы не выделяются. Вместо этого вычисляется TCP Cookie — хеш на основе секретного ключа сервера и четырёхэлементного кортежа (src IP, dst IP, src port, dst port). Cookie помещается в поле `seq` ответного SYN-ACK.
- При получении ACK от клиента модуль верифицирует Cookie. Если он корректен — пакет передаётся ядру на дальнейшую обработку (`XDP_PASS`), иначе — отбрасывается (`XDP_DROP`).
Весь перехват выполняется через фреймворк **XDP (eXpress Data Path)** — на уровне драйвера сетевой карты, до создания структуры `sk_buff`.
 
## Требования
 
- Ядро Linux версии 4.8 и выше
- Поддержка XDP драйвером сетевой карты
- `clang` и `llc` для компиляции BPF-объекта
- Утилита `ip` (пакет `iproute2`)
## Сборка
 
```bash
make
```
 
На выходе — объектный файл `xdp_filter.o`.
 
## Запуск и тестирование
 
Создать тестовый стенд (два сетевых пространства имён, соединённых через veth):
 
```bash
sudo ./stand up
```
 
Загрузить модуль и привязать к интерфейсу:
 
```bash
sudo ./stand attach
```
 
Просмотр отладочного вывода из журнала ядра:
 
```bash
sudo ./stand log
```
 
Остановить стенд:
 
```bash
sudo ./stand down
```
 
Для ручной привязки к произвольному интерфейсу:
 
```bash
ip link set dev eth0 xdp obj xdp_filter.o
```

## Алгоритм вычисления Cookie
 
Cookie вычисляется как цепочка CRC32-хешей по четырёхэлементному кортежу с фиксированным секретным ключом (`COOKIE_SECRET`):
 
```c
h = hash_crc32(src_ip, COOKIE_SECRET);
h = hash_crc32(dst_ip, h);
h = hash_crc32((src_port << 16) | dst_port, h);
cookie = client_seq + h;
```
 
Секретный ключ задаётся на этапе компиляции через макрос `COOKIE_SECRET` в `xdp_filter.c`.
 
## Ограничения
 
TCP Cookies несовместимы с TCP Options (Window Scale, SACK и др.): параметры из SYN-запроса не сохраняются, поскольку память под полуоткрытое соединение не выделяется.
