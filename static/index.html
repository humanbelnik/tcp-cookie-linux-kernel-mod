Курсовая работа по дисциплине "Компьютерные сети"

Разработка высокопроизводительного HTTP сервера для отдачи статического содержимого

Техническое задание

Функциональные требования

  • Поддержка GET и HEAD запросов
  • HTTP статусы: 200, 403, 404, 405
  • Файлы до 128 Мбайт
  • Мультиплексирование соединений
  • Встроенная HTML страница
  • Журналирование событий
  • Минимальные требования безопасности

Нагрузочное тестирование

  • Максимальное количество соединений
  • Скорость отдачи данных на соединение
  • Совокупная пропускная способность

Варианты архитектуры

thread pool + select()
prefork + select()
thread pool + pselect()
prefork + pselect()
thread pool + poll()
prefork + poll()
thread pool + epoll()
prefork + epoll()

Оформление работы

  • Расчётно-пояснительная записка на 12-32 листах А4
  • Презентация к курсовой работе на 8-16 слайдах

Вопросы и ответы

Какую операционную систему использовать?
Систему с поддержкой fork(), select(), pselect(), poll(), epoll() - любой современный Linux или *BSD системы
На каком языке программирования писать?
Языки с доступом к системным вызовам: C, C++, Swift, Rust. Высокоуровневые фреймворки запрещены
Какие библиотеки можно использовать?
Стандартная библиотека языка. Дополнительные библиотеки - только с разрешения руководителя
Какие есть материалы по теме?
Книга А.В. Столярова, The Linux Programming Interface, man-страницы Linux
